# Security Policy

## Do Not Post Private Data

Please do not post:

- Real customer names.
- Phone numbers, addresses, or emails.
- ERP exports.
- Delivery request documents.
- Real PDFs.
- API keys or tokens.
- Private model files.
- Screenshots containing private business data.

## Reporting Security Problems

If you find a security problem, do not open a public issue with sensitive details.

Instead, open a short issue that says:

```text
Security report available. Please contact me.
```

Do not include secrets or private data in the issue.

## Public Issue Safety

Before posting:

1. Replace real names with `Sample Customer`.
2. Replace phone numbers with `PHONE-REMOVED`.
3. Replace addresses with `Sample Address`.
4. Replace ERP IDs with `SAMPLE-001`.
5. Remove screenshots if they show private information.

## Maintainer Rule

Any pull request that includes private data should be rejected and removed from the public project history before further review.
